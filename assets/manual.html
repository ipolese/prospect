Prospector de Sites para Codex

O fluxo encontra leads, cria redesigns e apresenta os resultados primeiro em localhost. Hospedagem pública é opcional.

Fluxo

  1. $setup
  2. $prospectar
  3. $redesenhar
  4. $publicar em localhost
  5. $proposta somente após obter URL pública

Servidores separados

O dashboard administrativo usa 127.0.0.1:8765. O preview usa 127.0.0.1:8766 e expõe exclusivamente os arquivos permitidos dentro de sites/.

Nunca envie localhost ao cliente. Esse endereço só funciona na máquina que executa o servidor.

Publicação pública

FTP e o publicador automático recorrente foram desativados. A publicação segura utiliza SFTP, chave SSH, manifesto com hashes e confirmação explícita.

python3 publicador-seguro.py manifesto-publicacao.json --workspace .
python3 publicador-seguro.py manifesto-publicacao.json --workspace . \
  --confirm --host HOST --user USUARIO --key CAMINHO_DA_CHAVE

O primeiro comando apenas valida. O segundo publica depois que o usuário revisa a lista.

Credenciais

O projeto não armazena senha de hospedagem. Use chave SSH protegida por Keychain, Credential Manager, ssh-agent ou passphrase.
  • Não cole segredos no chat.
  • Não versione config, banco, leads ou contratos.
  • Não execute publicadores como administrador.
  • Use HTTPS válido na URL entregue ao cliente.

Dados pessoais

CPF/CNPJ, endereço, contratos e contatos devem permanecer locais, com acesso mínimo e conforme a LGPD. Nunca publique esses arquivos junto com os sites.
